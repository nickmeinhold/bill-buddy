## Project Overview

Bill Buddy is a personal finance app (similar to Rocket Money) built with Flutter. It tracks subscriptions, bills, budgets, and transactions with Firebase backend.

## Commands

```bash
# Run the app
flutter run

# Run on specific device (macos, chrome, or device ID)
flutter run -d macos

# Analyze code
flutter analyze

# Get dependencies
flutter pub get

# Clean build
flutter clean && flutter pub get

# Run tests
flutter test

# Build release
flutter build apk
flutter build ios
flutter build macos
```

## Architecture

### Tech Stack
- **State Management**: Riverpod (flutter_riverpod)
- **Routing**: GoRouter with auth guards
- **Backend**: Firebase (Auth + Firestore)
- **Charts**: fl_chart
- **Fonts**: Google Fonts (Inter)

### Key Patterns

**Riverpod Providers** - Each feature has:
- `{feature}Provider` - StreamProvider watching Firestore collection
- `{feature}ServiceProvider` - Service class with CRUD operations
- `firestoreProvider` - Shared Firestore instance (in subscriptions_provider.dart)

**Models** use Equatable and include:
- `fromMap(Map, String id)` factory for Firestore
- `toMap()` for serialization
- `copyWith()` for immutability
- Computed getters (e.g., `monthlyAmount`, `yearlyAmount` on SubscriptionModel)

**Routing** uses ShellRoute for persistent bottom navigation across 5 tabs: Dashboard, Transactions, Subscriptions, Budgets, Bills.

### Firestore Structure
```
users/{userId}/
├── subscriptions/{id}    # name, amount, frequency, nextBillingDate, category, status, notes
├── bills/{id}            # name, amount, dueDate, frequency, isPaid, remindDaysBefore, category, notes
├── budgets/{id}          # category, limit, spent, period, startDate
└── transactions/{id}     # merchantName, amount, date, category, isSubscription, notes
```

### Theme Colors
Semantic colors defined in `AppTheme`:
- `incomeColor` - green for positive amounts
- `expenseColor` - red for expenses
- `warningColor` - orange for budget warnings
- `subscriptionColor` - purple
- `billColor` - deep orange

## Firebase Setup

Requires `firebase_options.dart` generated by:
```bash
flutterfire configure
```

Firestore security rules needed:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```
